# 记忆中枢操作规则

## Episode 提取 (L1 → L2)

### 触发条件
1. Raw 层新增 > 20 条消息
2. 检测到会议结束标记

### 提取流程
1. 读取新增 Raw 消息
2. 调用 LLM 两阶段提取:
   - 阶段1: 提取候选事件
   - 阶段2: 分类标签 + 决定操作
3. 执行四操作写入:
   - ADD: 新建 Episode
   - UPDATE: 替代旧 Episode (旧版标记 superseded_by)
   - DELETE: 标记旧 Episode valid_until + reason
   - NOOP: 跳过不写入

### 四类标签定义
- **决策**: 团队达成的正式结论，必须有明确的采纳表述
- **待办**: 被分配了负责人的具体行动项
- **结论**: 对某问题的共识性认知，不一定有行动
- **疑问**: 明确提出但尚未有答案的问题

## Knowledge 合并 (L2 → L3)

### 触发条件
1. Episodes 层新增 > 10 条
2. 每日凌晨定时执行

### 合并流程
1. 读取新增 Episodes
2. 读取当前 Knowledge 四个文件
3. 调用 LLM 合并:
   - 新知识追加到对应分类
   - 被否决的旧知识标记失效时间
   - 已解决的疑问改为"已关闭"状态
4. 写入更新后的知识文件

### 四类知识文件
1. **技术决策库.md**: 所有正式技术决策 + 当时原因
2. **待跟进议题.md**: ★ HEARTBEAT 数据源，含状态:开放/已关闭
3. **项目规范.md**: 团队约定的工作方式和偏好
4. **人员职责.md**: 谁负责什么，联络方式
