# 项目助理操作规则

## 检索策略: 三层并行

用户提问后，同时向三层发出检索请求:
1. **L3 Knowledge**: 密度最高，优先命中
2. **L2 Episodes**: 补充上下文和过程细节
3. **L1 Raw**: 原始消息溯源

### 优先级规则
- L3 > L2 > L1
- L3 与 L2 冲突时，以 L3 为准
- valid_until=null 的记录优先于已失效记录
- UPDATE 操作的新记录优先于 superseded 的旧记录

## 四种交互模式

### 1. 被动问答
- 触发: 被 @提及 或 私聊
- 流程: 接收问题 → 并行检索 → 排序合并 → LLM 生成回答
- 回答格式: 答案 + 来源标注 (L1/L2/L3)

### 2. 主动推送
- 触发: 每日定时扫描待跟进议题.md
- 条件: 状态=开放 且距上次推送 >24小时
- 消息格式: 议题标题 + 负责人 + 截止日期

### 3. 定期摘要
- 日报: 每日 18:00，汇总当日 Episode
- 周报: 每周一 09:00，汇总上周 Episode
- 格式: 关键决策/新增待办/已完成/待解决

### 4. 决策追踪
- 触发: 命令 @assistant 决策时间线
- 输出: 按时间排列的所有技术决策
- 包含: 日期/决策内容/参与人/当前状态
